# 제품 요구사항 정의서 (PRD) - Premium 2048 웹 게임

본 문서는 네온 글래스모피즘 스타일과 부드러운 애니메이션을 적용한 프리미엄 웹 기반 2048 게임의 제품 요구사항을 정의합니다.

## 1. 개요 (Overview)
- **프로젝트 명**: Premium 2048 Game (Web-based)
- **목적**: 기존 2048 게임의 재미 요소를 극대화하고, 세련되고 현대적인 프리미엄 디자인(다크 모드, 네온 글래스모피즘, 부드러운 물리 기반 애니메이션)을 적용한 웹 기반 2048 게임 개발.
- **주요 타겟**: 브라우저에서 가볍게 게임을 즐기며 시각적인 만족감을 얻고자 하는 사용자.

## 2. 핵심 기능 요구사항 (Core Features)

### 2.1. 게임 보드 및 물리 로직
- **4x4 그리드 보드**: 게임은 기본적으로 4x4 크기의 격자판에서 진행됩니다.
- **방향 이동 및 병합 로직**:
  - 키보드 방향키(상, 하, 좌, 우) 또는 모바일 터치 스와이프 입력 시, 모든 타일이 해당 방향으로 이동합니다.
  - 동일한 숫자를 가진 두 타일이 충돌하면 하나로 병합되며 숫자가 2배가 됩니다. (예: 2 + 2 = 4, 4 + 4 = 8)
  - 한 번의 이동 주기에서 병합된 타일은 연속해서 다시 병합되지 않습니다.
  - 편의 기능 단축키 제공(숫자1 = 실행취소, 숫자2 = 랜덤 셔플, 숫자3 = 타일 스왑)
- **랜덤 타일 스폰**:
  - 타일이 이동한 후, 빈 셀 중 하나에 무작위로 새로운 타일(값은 2 또는 4, 2가 나올 확률 90%, 4가 나올 확률 10%)이 생성됩니다.
- **게임 상태 제어**:
  - **시작 (New Game)**: 보드를 비우고 랜덤 셀 두 곳에 초기 타일을 생성하여 게임을 시작합니다.
  - **진행 (Playing)**: 타일을 이동하고 점수를 쌓는 상태입니다.
  - **게임 오버 (Game Over)**: 빈 칸이 없고 인접한 칸 중 병합할 수 있는 동일한 숫자의 타일이 없는 경우 게임이 종료됩니다.
  - **승리 (Victory/2048)**: 하나의 타일의 값이 `2048`에 도달하면 승리 팝업이 표시됩니다. 사용자는 "계속 플레이하기"를 선택해 더 높은 점수에 도전하거나 게임을 재시작할 수 있습니다.

### 2.2. 편의 기능
- **1.실행 취소 (Undo)**:
  - 직전 1단계 상태로 되돌리는 Undo 기능을 제공합니다. (직전에 획득한 점수의 50% 차감)
- **2.랜덤 셔플 (Random Shuffle)**:
  - 현재 타일의 위치를 유지한 채 숫자만 랜덤하게 뒤섞는 기능을 제공합니다. (직전에 획득한 점수의 30% 차감)
- **3.타일 스왑 (Tile Swap)**:
  - 사용자가 선택한 두 개의 타일의 위치를 서로 바꾸는 기능을 제공합니다. (직전에 획득한 점수의 70% 차감)
- **자동 저장 및 재개 (Auto-save & Resume)**:
  - 플레이어가 브라우저를 닫거나 새로고침하더라도 이전 게임 상태(그리드 배열, 점수, 최고 점수 등)가 브라우저의 `localStorage`에 자동 저장되어 이어서 플레이할 수 있습니다.
- **점수 관리**:
  - **현재 점수 (Current Score)**: 타일이 병합될 때마다 병합된 타일의 값이 현재 점수에 합산됩니다.
  - **최고 점수 (Best Score)**: 역대 최고 점수를 `localStorage`에 저장하여 유지합니다. 현재 점수가 최고 점수를 넘어서는 즉시 갱신됩니다.
  - **점수 랭킹 (Score Ranking)**: 오늘 획득한 점수를 기준으로 순위를 매겨 `localStorage`에 저장하여 유지합니다.

## 3. UI/UX 디자인 요구사항 (UI/UX & Aesthetics)

- **프리미엄 비주얼 (Rich Aesthetics)**:
  - **테마**: 몽환적이고 포근한 분위기를 주는 **솜사탕 그라데이션 라이트 테마** 및 느리게 흘러가는 구름 애니메이션 배경 적용.
  - **스타일**: 부드러운 라운딩 처리와 따뜻한 화이트 크림 톤의 **소프트 글래스모피즘(Soft Glassmorphism)** 스타일의 컨테이너 설계.
  - **타일 컬러**: 눈이 편안하고 아기자기한 **동화풍 파스텔 컬러** (베이비 핑크, 소프트 블루, 민트, 멜로우 옐로우, 소프트 코랄 등) 매핑. 2048 이상 타일은 은은하고 부드럽게 반짝이는 소프트 펄스(Soft Pulse) 효과 적용.
  - **폰트**: Google Fonts에서 Outfit 또는 Inter 폰트를 적용하여 가독성과 현대적인 느낌을 강조.
  - **편의 기능 버튼**: 
    - 실행 취소 (Undo)
    - 랜덤 셔플 (Random Shuffle)
    - 타일 스왑 (Tile Swap)

- **부드러운 애니메이션 (Micro-animations & Transitions)**:
  - **타일 이동**: CSS `transform` 및 `transition`을 사용해 타일이 부드럽게 미끄러지듯 이동하는 애니메이션.
  - **타일 병합**: 두 타일이 합쳐지는 순간 크기가 살짝 커졌다가 제자리로 돌아오는 바운스 효과 적용.
  - **타일 생성**: 새로운 타일이 생성될 때 투명도(opacity)와 스케일(scale)이 `0`에서 `1`로 부드럽게 커지는 효과.
  - **점수 획득 이펙트**: 점수가 추가될 때마다 "+X" 형태의 텍스트가 점수판 위로 둥둥 떠다니며 서서히 사라지는(Floating popup) 애니메이션.
- **반응형 및 입력 지원**:
  - **모바일/태블릿**: CSS Flexbox/Grid와 미디어 쿼리를 사용하여 다양한 디바이스 화면 크기에 최적화된 레이아웃을 제공합니다. 모바일에서는 스와이프 제스처를 위한 `Touch Events` 처리.
  - **데스크톱**: 방향키(`ArrowUp`, `ArrowDown`, `ArrowLeft`, `ArrowRight`) 및 WASD 입력 지원.

## 4. 기술 스택 및 비기능 요구사항
- **개발 환경 및 스택**:
  - 마크업: HTML5 (Semantic Tag 구조)
  - 스타일: Vanilla CSS (CSS 변수를 사용한 컬러 테마 제어)
  - 스크립트: Vanilla JavaScript (ES6+ 문법, 객체 지향 또는 모듈형 설계)
- **성능 및 접근성**:
  - CSS 하드웨어 가속(`will-change`)을 활용하여 모바일 기기에서도 초당 60프레임(FPS)의 부드러운 애니메이션 유지.
  - 마우스 클릭, 터치, 키보드 입력의 딜레이(latency)를 최소화하여 빠른 조작감 제공.
  - 웹 표준을 준수하고 브라우저 호환성(Chrome, Safari, Firefox, Edge 등) 보장.

## 5. 구현 마일스톤 (Milestones)
1. **1단계**: HTML 구조 설계 및 CSS 변수를 활용한 네온 글래스모피즘 테마 스타일링.
2. **2단계**: JavaScript 2048 핵심 물리 엔진 구현 (이동, 병합, 스폰, 상태 판단).
3. **3단계**: CSS 트랜지션을 활용한 부드러운 이동/병합/생성 애니메이션 구현 및 연동.
4. **4단계**: 로컬스토리지 연동(최고점수, 상태 복구), 실행 취소(Undo) 구현, 모바일 터치 이벤트 연동.
5. **5단계**: 최종 디자인 폴리싱 및 다양한 브라우저/디바이스 테스트 및 검증.
